📐 원리

사인파 호흡 곡선

밝기는 사인파(sine wave)를 따라 30% ~ 100% 사이를 부드럽게 오르내립니다. 1왕복 600ms (25fps)로, 자연스러운 호흡 리듬을 모방합니다. 

double calc_brightness(int step) {
    double t = (double)(step % CYCLE_STEPS) / CYCLE_STEPS * 2.0 * M_PI;
    double s = sin(t);
    // 0.30 ~ 1.00 사이를 부드럽게 오르내림
    return 0.30 + (1.00 - 0.30) * (s + 1.0) / 2.0;
  }

ANSI TrueColor

#FFCC3D (R=255, G=204, B=61) 의 RGB 값을 밝기 계수로 스케일링합니다. \033[38;2;R;G;Bm ANSI TrueColor 이스케이프 시퀀스로 출력됩니다. 대부분의 현대 터미널(iTerm2, Windows Terminal, kitty, alacritty)에서 지원합니다. 

  printf("\r\033[38;2;%d;%d;%dm{◆}\033[0m   ", r, g, b);
  fflush(stdout);

단일 쓰레드 select() 루프

별도의 쓰레드 없이, select() 시스템 콜로 40ms 타임아웃을 설정합니다. 타임아웃마다 호흡을 갱신하고, 데이터가 도착하면 풀밝기로 고정 후 청크를 처리합니다. 

  // 40ms select 타임아웃 → 호흡 갱신
  struct timeval tv = {0, 40000};
  int sret = select(fd + 1, &rfds, NULL, NULL, &tv);
  if (sret == 0) { /* timeout → breathe */
      breath_tick();
      continue;
  }
  // Data ready → lock, then BIO_read
  breath_lock();

🧠 설계 결정

문자 교체 ❌ → 색상 호흡 ✅
◇ → ◈ → ◆ 같은 유니코드 문자 교체는 깜빡임(blink) 느낌이 있습니다. 같은 {◆} 를 유지하면서 밝기만 변화시키는 방식이 더 자연스럽습니다. { 와 } 는 고정점(invariant)으로, 변화하지 않습니다.

멀티쓰레드 ❌ → select() ✅
현재는 단일 쓰레드지만, select() 기반이므로 향후 멀티쓰레드 전환 시에도 usleep() 타이머로 자연스럽게 업그레이드됩니다.
